# Project Summary

The "Strength Tracker" is a progressive web application (PWA) designed specifically for strength training enthusiasts.
Built with React and Tailwind CSS, the application provides a streamlined interface for logging exercises, tracking
workout history, and applying the principle of progressive overload.

Key features include a focus-mode training interface, readiness-based dynamic workout adjustments (evaluating CNS state
to suggest training volume), automated weight progression based on past records, and localStorage persistence for
seamless session recovery. The design incorporates a dark-themed Glassmorphism aesthetic, ensuring a modern and
responsive user experience across devices.

# Project Module Description

The application architecture is divided into several key modules:

1. **Dashboard**: The main hub displaying overall training statistics (total workouts, tonnage) and a chronological
   history of past sessions.
2. **Readiness Screen**: Evaluates the user's pre-workout state (scale 1-5) to dynamically adjust the number of sets and
   automatically calculates target weights based on historical data.
3. **Workout Screen**: A focus-mode interface displaying one exercise at a time, complete with visual references (
   images), past performance data, dynamic weight suggestions (+2.5 kg for optimal readiness), and sequential
   navigation.
4. **Modals**: Handles edge cases such as adding custom exercises on the fly and validating early exits to prevent
   accidental data loss.

# File Description Inventory

* **index.html**: Sets up the HTML structure for the app and mounts the React application.
* **package.json**: Contains project dependencies, scripts for development, building, and linting.
* **vite.config.js**: Configures Vite for compiling the React application and managing static asset paths.
* **public/images/**: Directory serving static image assets for exercise visualization.
* **manifest.json / sw.js**: Enable PWA features, allowing the app to be installed and run locally on mobile devices.
* **src/index.css**: Contains global CSS variables and Tailwind utilities.
* **src/main.jsx**: Bootstraps the application environment.
* **src/App.jsx**: The top-level orchestrator. Manages screen transitions and processes the final workout payload into
  the history ledger.
* **src/components/Dashboard.jsx**: Renders lifetime stats and previous workout summaries.
* **src/components/ReadinessScreen.jsx**: Intercepts the user before a workout to gather readiness metrics and select
  target muscle groups.
* **src/components/WorkoutScreen.jsx**: The core tracking interface featuring form inputs, exercise images, historical
  weight retrieval, and exercise navigation.
* **src/utils/workoutData.js**: Acts as the local database schema. Contains the `MUSCLE_GROUPS` object and logic for
  generating personalized workout arrays based on readiness and past history.
* **src/utils/storage.js**: Abstracts the browser's `localStorage` API, handling data serialization/deserialization for
  active sessions and historical ledgers.

# Technology Stack

* **Frontend**: React.js, Tailwind CSS, ES6+ JavaScript, Lucide Icons
* **Build & Compilation**: Vite, ESLint
* **PWA Support**: Native Service Workers, Web App Manifest
* **Deployment**: GitHub Actions (CI/CD) to GitHub Pages

# Usage

To set up and run the application locally:

1. Install dependencies:

```bash
npm install

```

2. Start the local development server:

```bash
npm run dev

```

3. Build the project for production:

```bash
npm run build
```

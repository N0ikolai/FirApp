# Project Summary

**FirApp** (formerly "Strength Tracker") is a progressive web application (PWA) designed specifically for strength training enthusiasts. Built with React and Tailwind CSS, the application provides a streamlined interface for logging exercises, tracking workout history, and applying the principle of progressive overload.

Key features include a focus-mode training interface, readiness-based dynamic workout adjustments (evaluating CNS state to suggest training volume), automated weight progression based on post-workout difficulty feedback (+2.5 kg / -2.5 kg auto-regulation), an automated rest timer, quick-input presets for sweaty hands, minimalist record-breaking confetti celebrations, tactile haptic feedback, a weekly interactive schedule planner, and localStorage persistence for seamless session recovery. The design incorporates a dark-themed Glassmorphism aesthetic, ensuring a modern and responsive user experience across devices.

# Project Module Description

The application architecture is divided into several key modules:

1. **Dashboard**: The main hub displaying overall training statistics (total workouts, tonnage), a chronological history of past sessions, an interactive weekly workout schedule planner, and a randomized daily fitness tip ("Reality check") card.
2. **Readiness Screen**: Evaluates the user's pre-workout state (scale 1-5) to dynamically adjust the number of sets. It also intercepts historical data to automatically adjust target weights based on the user's difficulty rating from the previous session.
3. **Workout Screen**: A focus-mode interface displaying one exercise at a time. Features include visual references (images), past performance data, quick adjustment presets (+/- buttons for weight and reps), an integrated rest timer with skip/+30s extensions, sequential navigation, tactile haptic feedback via the Vibration API, and a post-workout difficulty evaluation modal.
4. **Modals & Settings**: Handles edge cases such as adding custom exercises on the fly, validating early exits to prevent accidental data loss, clearing data, and toggling system settings like haptic feedback.

# File Description Inventory

- **index.html**: Sets up the HTML structure for the app, mounts the React application, and strictly defines PWA asset paths.
- **package.json**: Contains project dependencies, scripts for development, building, and linting.
- **vite.config.js**: Configures Vite for compiling the React application and managing static asset paths.
- **public/**: Directory serving static image assets, valid PWA icons (192.png, 512.png), and web manifests.
- **manifest.json / sw.js**: Enable robust PWA features, allowing the app to be installed natively on Android/iOS devices.
- **src/index.css**: Contains global CSS variables and Tailwind utilities.
- **src/main.jsx**: Bootstraps the application environment.
- **src/App.jsx**: The top-level orchestrator. Manages screen transitions and processes the final workout payload into the history ledger, triggering a confetti celebration if a new tonnage record is set.
- **src/components/Dashboard.jsx**: Renders lifetime stats, previous workout summaries, an interactive 7-day schedule, fitness tips, and global app settings.
- **src/components/ReadinessScreen.jsx**: Intercepts the user before a workout to gather readiness metrics, select target muscle groups, and parse previous workout difficulties to apply progressive overload mathematics.
- **src/components/WorkoutScreen.jsx**: The core tracking interface featuring form inputs, exercise images, historical weight retrieval, quick adjustment presets, an automated rest timer, haptic vibration triggers, and a post-workout evaluation prompt.
- **src/utils/workoutData.js**: Acts as the local database schema. Contains the `MUSCLE_GROUPS` object and logic for generating personalized workout arrays based on readiness.
- **src/utils/storage.js**: Abstracts the browser's localStorage API, handling data serialization/deserialization for active sessions, historical ledgers, and user preferences (like vibration settings).

# Technology Stack

- **Frontend**: React.js, Tailwind CSS, ES6+ JavaScript, Lucide Icons
- **Browser APIs**: Web App Manifest, Service Workers, Navigator Vibration API
- **Libraries**: `canvas-confetti` (for visual celebrations)
- **Build & Compilation**: Vite, ESLint
- **Deployment**: GitHub Actions (CI/CD) to GitHub Pages

# Usage

To set up and run the application locally:

1. Install dependencies:
   ```bash
   npm install
   ```

2. Start the local development server:
   ```bash
   npm run dev
   ```

3. Build the project for production:
   ```bash
   npm run build
   ```
